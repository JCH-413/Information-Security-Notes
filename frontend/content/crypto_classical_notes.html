01

Key Terminology & Cryptographic Systems

📖 Core Vocabulary

  • Plaintext — the original message.
  • Ciphertext — the coded message.
  • Key — information used in encryption/decryption, known only to sender/receiver.
  • Encipher (encrypt) — converting plaintext to ciphertext using a key.
  • Decipher (decrypt) — recovering plaintext from ciphertext using a key.
  • Cryptography — the study of encryption principles, methods, and designs.
  • Cryptanalysis (code-breaking) — the study of principles/methods of deciphering ciphertext (without the key).

🧭 Three Ways to Classify Cryptographic Systems

1. By the operation used to transform plaintext into ciphertext:

  • Substitution — each element in the plaintext is mapped into another element.
  • Transposition — the elements in the plaintext are re-arranged.

2. By the number of keys used:

  • Symmetric (private-key) — both sender and receiver use the same key.
  • Asymmetric (public-key) — sender and receiver use different keys.

3. By how plaintext is processed:

  • Block cipher — inputs are processed one block at a time, producing a corresponding output block.
  • Stream cipher — inputs are processed continuously, producing one element at a time (bit or byte).

🔑 Requirements for Secure Symmetric Encryption

Two requirements for the secure use of symmetric encryption:

  • A strong encryption algorithm.
  • A secret key known only to sender / receiver.
Y = EK(X) encryption: X = plaintext, Y = ciphertext
X = DK(Y) decryption recovers the plaintext

We assume the encryption algorithm is known to all parties (Kerckhoffs's principle); this implies a secure channel is needed to distribute the key.

📖 Monoalphabetic vs Polyalphabetic

  • Monoalphabetic — replaces each letter with another single letter/symbol regardless of its position; every occurrence of a letter is replaced by the same letter each time. The additive, shift, Caesar, and affine ciphers are all monoalphabetic.
  • Polyalphabetic — uses multiple "alphabets" to encipher. If two letters are the same in the ciphertext, it does not mean they decipher to the same plaintext letter. Playfair, Hill, and Vigenère are polyalphabetic.
03

Caesar Cipher

📖 What It Is

  • The earliest known substitution cipher.
  • First attested use in military affairs.
  • Replaces each letter by the 3rd letter on (a fixed shift).

Example with shift 3: meet me after the toga party → phhw ph diwhu wkh wrjd sduwb.

⚙️ The Transformation

With the letters numbered a=0, b=1, …, z=25, the Caesar cipher is just modular addition by the key k:

c = E(k, p) = (p + k) mod 26 encryption
p = D(k, c) = (c − k) mod 26 decryption

For the classic shift of 3: plain a b c … x y z maps to cipher D E F … A B C. Diagrammatically: sender "Add Kₑ = 3", receiver "Subtract K_d = 3" — e.g. DEAR DEAR FRIEND → GHDU GHDU IULHQG → back to DEAR DEAR FRIEND.

🔓 Cryptanalysis of the Caesar Cipher

The Caesar cipher is trivially broken by brute force because:

  • There are only 25 possible keys (A maps to B, C, … Z).
  • You can simply try each one in turn.

A brute-force search succeeds when three conditions hold:

  1. The encryption and decryption algorithms are known.
  2. There are only 25 keys to try.
  3. The language of the plaintext is known and easily recognizable.

Worked brute-force: ciphertext PHHW PH DIWHU WKH WRJD SDUWB tried against all 25 keys reveals key 3 → meet me after the toga party (the only one that reads as English). Another exercise from the slides: break GCUA VQ DTGCM and GCUA VQ DTGCM.

⚖️ Brute-Force Feasibility (the general lesson)

  • In most networking situations we can assume the algorithms are known.
  • What makes brute force impractical is an algorithm employing a large number of keys.
  • Example: Triple DES uses a 168-bit key, giving a key space of 2¹⁶⁸ > 3.7 × 10⁵⁰ possible keys.

Affine Cipher (with worked maths)

📖 Definition

An encipherment scheme of the form below is called an affine cipher:

E(x) = (a·x + b) mod 26
x = numeric value of the plaintext letter; a, b are chosen integers

It generalizes the Caesar cipher (which is just the case a = 1). The letter↔number mapping is the standard one:

ABCDEFGHIJKLM
0123456789101112
NOPQRSTUVWXYZ
13141516171819202122232425

⚙️ Deriving the Decryption Function

If y = E(x) = (a·x + b) mod 26, we solve for x in terms of y:

y ≡ a·x + b (mod 26)
y − b ≡ a·x (mod 26)
a·x ≡ (y − b) (mod 26)
multiply both sides by a⁻¹ (mod 26):
x ≡ a⁻¹·(y − b) (mod 26)

So the decipherment function is:

E⁻¹(y) = a⁻¹·(y − b) mod 26

The catch: a must have a modular inverse mod 26, which only happens when gcd(a, 26) = 1. That is why a = 5 works (gcd(5,26)=1) but a = 13 would not.

🔓 Monoalphabetic Cipher Security

  • A general monoalphabetic substitution has 26! ≈ 4 × 10²⁶ keys.
  • With so many keys you might think it is secure — but that would be WRONG.
  • The problem is language characteristics: a monoalphabetic cipher does not change relative letter frequencies.

Frequency attack: calculate letter frequencies in the ciphertext and compare them against known English values (E ≈ 12.7%, T ≈ 9.1%, A ≈ 8.2% are the most common; J, Q, X, Z are rarest). The peaks line up and the key falls out.

Playfair Cipher

📖 What It Is

  • Even the large key count of a homophonic cipher does not provide security; one improvement is to encrypt multiple letters at once.
  • The Playfair cipher encrypts digraphs (pairs of letters) and was invented by Charles Wheatstone in 1854 (named after Lord Playfair who promoted it).

🔢 The 5×5 Key Matrix

Build a 5×5 matrix from a keyword: fill in the keyword's letters (dropping duplicates), then fill the rest with the remaining alphabet. I and J share a cell (25 cells for 26 letters). Example with keyword MONARCHY:

MONAR
CHYBD
EFGI/JK
LPQST
UVWXZ

📏 Encryption Rules (plaintext taken two letters at a time)

  1. If a pair is a repeated letter, insert a filler like 'X' between them.
  2. If both letters are in the same row, replace each with the letter to its right (wrapping from the end back to the start).
  3. If both letters are in the same column, replace each with the letter below it (wrapping from bottom back to top).
  4. Otherwise (a rectangle), replace each letter with the one in its own row but in the column of the other letter of the pair.

Hill Cipher

📖 What It Is

  • A polyalphabetic cipher based on matrix multiplication, and another historical cipher subject to known-plaintext attacks.
  • Take K, an M×M key matrix representing a system of linear equations.
  • Ciphertext C and plaintext p are column vectors (here written as 1×M / M×1 matrices).

⚙️ Encryption & Decryption

C = K·p (mod 26) encryption: multiply the key matrix by the plaintext vector

For a 3×3 example, each ciphertext component is a linear combination:

C₁ = (k₁₁·p₁ + k₁₂·p₂ + k₁₃·p₃) mod 26
C₂ = (k₂₁·p₁ + k₂₂·p₂ + k₂₃·p₃) mod 26
C₃ = (k₃₁·p₁ + k₃₂·p₂ + k₃₃·p₃) mod 26

Decryption requires the key matrix K to be invertible mod 26:

p = K⁻¹·C (mod 26) K⁻¹ (mod 26) must exist

🔓 Weakness

The Hill cipher is subject to a known-plaintext attack: if the attacker can choose the plaintext and have it enciphered for them, they can recover the key matrix by solving the resulting linear equations.

Vigenère Cipher & the One-Time Pad

📖 The Vigenère Cipher

  • The simplest polyalphabetic substitution cipher — effectively multiple Caesar ciphers.
  • The key is multiple letters long: K = k₁ k₂ … k_d.
  • The iᵗʰ letter of the key specifies the iᵗʰ alphabet (shift) to use; use each alphabet in turn and repeat from the start after d letters.
  • Decryption simply works in reverse.

⚙️ The Equations

Ci = (pi + k(i mod m)) mod 26 encryption
pi = (Ci − k(i mod m)) mod 26 decryption

🧮 Worked Example — key "deceptive"

Encrypting wearediscoveredsaveyourself with the repeating key deceptive:

key: deceptivedeceptivedeceptive
plaintext: wearediscoveredsaveyourself
ciphertext: ZICVTWQNGRZGVTWAVZHCQYGLMGJ

Numerically, the first few columns: key 3,4,2,4,15,19,8,21,4…; plaintext 22,4,0,17,4,3,8,18,2…; giving ciphertext 25,8,2,21,19,22,16,13,6… → Z,I,C,V,T,W,Q,N,G…

Worked: w(22)+d(3)=25→Z; e(4)+e(4)=8→I; a(0)+c(2)=2→C; and so on, the key recycling every 9 letters.

🛡️ Security of the Vigenère Cipher

  • It provides multiple ciphertext letters for each plaintext letter, so letter frequencies are obscured.
  • But they are not totally lost — the frequency distribution still leaks structure (this is what Kasiski/Friedman attacks exploit).

🔐 The One-Time Pad (OTP)

  • If a truly random key as long as the message is used, the cipher is secure — this is the One-Time Pad.
  • It is unbreakable because the ciphertext bears no statistical relationship to the plaintext: for any plaintext and any ciphertext, there exists a key mapping one to the other.
  • The key can only be used once.
  • Drawbacks: serious problems in generation and safe distribution of the key; cannot be used extensively, but it does provide perfect secrecy.

🧩 OTP Security — the intuition

The same ciphertext can decrypt to completely different sensible messages under different keys, so an attacker learns nothing. From the slides, the ciphertext ANKYODKYUREPFJBYOJDSPLREYIUNOFDOIUERFPLUYTS:

  • with one key → mr mustard with the candlestick in the hall
  • with another key → miss scarlet with the knife in the library

Because every plaintext of the right length is equally possible, the attacker cannot tell which one is real — that is the essence of perfect secrecy.

Transposition Ciphers — Concept

📖 What Transposition Means

  • A very different kind of mapping: instead of substituting letters, you perform a permutation on the plaintext letters.
  • These hide the message by rearranging letter order without altering the actual letters used.
  • A pure transposition cipher can be recognized because it has the same letter-frequency distribution as the original plaintext (the letters are the same, just moved).

⚠️ Problem with Transposition

  • A pure transposition cipher is easily recognized due to its unchanged letter frequencies.
  • It can be made significantly more secure by performing more than one stage of transposition.

🔗 Product Ciphers (combining two approaches)

  • Ciphers using substitution or transposition alone are not secure because of language characteristics.
  • So we use several ciphers in succession. But note:
    • Two substitutions just make a more complex substitution.
    • Two transpositions just make a more complex transposition.
    • A substitution followed by a transposition makes a new, much harder cipher.
  • This is the bridge from classical to modern ciphers (e.g. DES alternates substitution and permutation).

Rail Fence & Row Transposition

📖 Rail Fence Cipher

  • Write the message letters out diagonally over a number of rows.
  • Then read off the cipher row by row.

Example — message meet me after toga party written on 2 rails:

m . e . m . a . t . r . t . g . p . r . y
. e . t . e . f . e . t . o . a . a . t .

Reading row 1 then row 2 gives the ciphertext:

Ciphertext: MEMATRHTGPRYETEFETEOAAT

📖 Row (Columnar) Transposition Cipher

  • A more complex transposition. Write the message letters out in rows over a specified number of columns.
  • Then reorder the columns according to a key before reading off the rows (read column by column in key order).

Example — key 4 3 1 2 5 6 7, plaintext attackpostponeduntil (padded):

Key→4312567
row1attackp
row2ostpone
row3duntilt
row4woamxyz

Read the columns in the order given by the key (column labelled 1 first, then 2, 3, 4…):

Ciphertext: TTNAAPTMTSUOAODWCOIXKNLYPETZ

The key numbers tell you the order in which to read the columns off — the smallest key digit's column is read first.

Rotor Machines

📖 What They Are

  • Before modern ciphers, rotor machines were the most common complex ciphers in use.
  • Used in WW2 — the German Enigma, the Allied Hagelin, and the Japanese Purple.
  • They implemented a very complex, varying substitution cipher.
  • They used a series of cylinders, each giving one substitution, which rotated and changed after each letter was encrypted (like an odometer).

🔢 Why They Were Strong

3 cylinders → 26³ = 17,576 alphabets
5 cylinders → 26⁵ = 11,881,376 alphabets

The diagram of a 3-rotor machine shows a fast rotor, medium rotor, and slow rotor; after each keystroke the fast rotor advances one position, periodically stepping the next rotor — so the substitution alphabet differs for every letter.

🔓 Rotor Machine Security

  • A given setting of a 5-rotor machine is equivalent to a Vigenère cipher with key length 11,881,376 (26⁵) — a formidable cryptanalytic challenge.
  • A frequency-analysis attacker would face the equivalent of over 11 million monoalphabetic ciphers.
  • Needing ~50 letters per monoalphabetic cipher to solve it would require a ciphertext of over half a billion letters.

The Hagelin rotor machine (a portable mechanical device with stepping cylinders) is the physical example shown in the lecture.
