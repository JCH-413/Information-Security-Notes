RSA Introduction & Background

📖 What Is RSA?

RSA stands for Rivest, Shamir, and Adleman — the three inventors. It was developed at MIT in 1978 and is one of the most widely used public-key cryptographic algorithms in the world.

  • RSA is a block cipher algorithm — it operates on fixed-size blocks of data
  • Plaintext and ciphertext are integers in the range 0 to n
  • Typical block size: 1024 bits (historically); modern use: 2048 or 4096 bits
  • It is an asymmetric (public-key) cryptosystem — uses two different keys

🔑 The Two Keys in RSA

Public Key = (e, n)
Private Key = (d, n)

Encryption: C = Mᵉ mod n
Decryption: M = Cᵈ mod n
  • e = public exponent (shared openly)
  • d = private exponent (kept secret)
  • n = modulus (part of both keys)

🧱 The Core Security Principle

RSA's security relies on the Integer Factorization Problem:

  • Multiplying two large prime numbers p and q to get n = p × q is computationally easy
  • Factoring n back into p and q (when n is very large) is computationally infeasible with current technology

This asymmetry — easy to do, hard to undo — is the mathematical trapdoor RSA is built on.

🧠 Beginner Explanation

Imagine multiplying two huge prime numbers is like mixing two paints to get a unique color — easy to mix, but impossible to tell from the result what the original colors were. RSA uses this one-way mixing property to create a lock (public key) that anyone can use, but only the key-owner (private key) can unlock.

⚠ Common Misconceptions

  • RSA is NOT used to encrypt large files — it's too slow. It's used for key exchange and digital signatures
  • Both n values (public and private key) are the SAME n = p × q
  • Larger key size = more security BUT also much slower computation
  • RSA key length refers to the size of n, not e or d individually

Key Generation — Theory & Steps

📖 Key Generation Overview

Key generation is performed by the receiver (the person who wants to receive encrypted messages). They generate both their public key (which they share) and their private key (which they keep secret).

🔢 Key Generation — Step by Step

Step 1 — Choose Two Large Primes

Select two distinct large prime numbers p and q at random.

p and q must be large, distinct prime numbers

In real RSA: p and q are hundreds of digits long. In examples, small numbers are used for illustration.

Step 2 — Compute the Modulus n

Calculate the modulus n = p × q. This is shared as part of both the public and private keys.

n = p × q

The length of n in bits is the "key length" of RSA (e.g., 2048-bit RSA means n has 2048 bits).

Step 3 — Compute Euler's Totient φ(n)

Calculate φ(n) = (p−1)(q−1). This value is kept secret after key generation.

φ(n) = (p−1)(q−1)

φ(n) counts how many integers from 1 to n are coprime with n. This is only easy to compute if you know p and q — which is why factoring n would break RSA.

Step 4 — Choose Public Exponent e

Select an integer e (the public exponent) such that:

1 < e < φ(n)
GCD(e, φ(n)) = 1

e and φ(n) must be coprime (share no common factor other than 1). Common values: e = 3, 17, or 65537.

Step 5 — Compute Private Exponent d

Find d such that it is the modular multiplicative inverse of e modulo φ(n):

d × e ≡ 1 (mod φ(n))
Or equivalently: d × e = 1 + k × φ(n) for some integer k

This is calculated using the Extended Euclidean Algorithm (see Section 6).

Step 6 — Publish Public Key, Keep Private Key Secret
Public Key: KU = {e, n}
Private Key: KR = {d, n} (also destroy p, q, φ(n) after use)

RSA Encryption

📖 Encryption — Definition

RSA encryption is performed by the sender using the receiver's public key. The sender converts their message into a number and applies the encryption formula.

🔒 Encryption Formula

C = Mᵉ mod n

Where:
M = plaintext message (as a number, M < n)
e = public exponent
n = modulus
C = ciphertext output
Important: M must be less than n. If the message is larger than n, it must be broken into blocks.

📝 Encryption Steps

  1. Obtain the receiver's public key (e, n)
  2. Convert message to a number M (e.g., ASCII encoding)
  3. Ensure M < n (break into blocks if needed)
  4. Compute: C = Mᵉ mod n
  5. Send ciphertext C to receiver

🌍 Real-World Context

In practice, text is converted to numbers using schemes like ASCII or Unicode. For example, the letter 'A' = 65 in ASCII. The entire message is treated as a large number (or broken into blocks if too large).

Modern systems use RSA only to encrypt a symmetric key (like an AES key), not the full message — because RSA is slow for large data.

  • MCQ: Who performs encryption in RSA? → The sender, using receiver's public key
  • MCQ: Which key is used for RSA encryption? → Receiver's public key (e, n)
  • Short: What constraint must the plaintext M satisfy before encryption? → M < n

04

RSA Decryption

📖 Decryption — Definition

RSA decryption is performed by the receiver using their own private key (d, n) to recover the original plaintext M from the ciphertext C.

🔓 Decryption Formula

M = Cᵈ mod n

Where:
C = ciphertext received
d = private exponent (secret)
n = modulus
M = recovered plaintext

📝 Decryption Steps

  1. Obtain private key parameters (d, n)
  2. Receive ciphertext C
  3. Perform modular exponentiation: M = Cᵈ mod n
  4. Recover the original plaintext number M
  5. Convert M back to the original message (e.g., ASCII)

Cᵈ mod n is an extremely large calculation, but it can be done efficiently using modular exponentiation (square-and-multiply algorithm) — so "impossibly large" numbers in examples are computed efficiently in practice.

Full Worked Example — Alice & Bob

📖 Scenario

Alice wants to send a secret message to Bob. Bob generates the key pair; Alice encrypts with Bob's public key; Bob decrypts with his private key.

Step 1 — Key Generation (Bob's Job)

Choose two distinct primes:

p = 61, q = 53

Compute n:

n = p × q = 61 × 53 = 3233

→ n = 3233. This is the modulus and is part of BOTH keys. Its bit-length is the "key length."

Step 2 — Compute φ(n)
φ(n) = (p−1)(q−1) = (61−1)(53−1) = 60 × 52 = 3120

→ φ(n) = 3120. This is kept secret.

Step 3 — Choose Public Exponent e

Pick e such that 1 < e < 3120 and GCD(e, 3120) = 1.

e = 17

Verify: GCD(17, 3120) = 1 ✓ (17 is prime and does not divide 3120)

→ Public exponent e = 17

Step 4 — Compute Private Exponent d

Find d such that: (d × 17) mod 3120 = 1

Using the Extended Euclidean Algorithm (detailed in Section 6):

d = 2753

Verification: 2753 × 17 = 46801. Then: 46801 mod 3120 = 1 ✓

→ Private exponent d = 2753

Step 5 — Bob's Final Keys
Public Key = (e=17, n=3233) ← shared with Alice
Private Key = (d=2753, n=3233) ← Bob keeps secret
Step 6 — Encryption (Alice's Job)

Alice's message is the number M = 65 (e.g., ASCII for 'A'). Note: 65 < 3233 ✓

C = Mᵉ mod n = 65¹⁷ mod 3233

Using modular exponentiation:

C = 2790

→ Alice sends ciphertext C = 2790 to Bob.

Step 7 — Decryption (Bob's Job)

Bob receives C = 2790 and uses his private key:

M = Cᵈ mod n = 2790²⁷⁵³ mod 3233

Using modular exponentiation:

M = 65

→ Bob successfully recovers the original message M = 65. ✅

Extended Euclidean Algorithm — Finding d

📖 What Is It?

The Extended Euclidean Algorithm is a method to find the modular multiplicative inverse — i.e., to solve the equation d × e ≡ 1 (mod φ(n)). It extends the basic Euclidean Algorithm for GCD to also find the coefficients d and k that satisfy e × d + φ(n) × k = 1.

🔢 Problem to Solve

Given: e = 17, φ(n) = 3120
Find: d such that 17 × d ≡ 1 (mod 3120)
Equivalently: find d and k such that 17 × d + 3120 × k = 1

📐 Step 1 — Run the Euclidean Algorithm (Find GCD)

Repeatedly divide and record remainders:

3120 = 183 × 17 + 9    (3120 ÷ 17 = 183, remainder 9)
17 = 1 × 9 + 8        (17 ÷ 9 = 1, remainder 8)
9 = 1 × 8 + 1        (9 ÷ 8 = 1, remainder 1)
8 = 8 × 1 + 0        (8 ÷ 1 = 8, remainder 0 → STOP)

GCD(17, 3120) = 1 ✓ (as expected — they are coprime)

📐 Step 2 — Back-Substitute to Find d

Work backwards through the equations:

From: 9 = 1 × 8 + 1 → 1 = 9 − 1 × 8

Substitute 8 = 17 − 1 × 9:
1 = 9 − 1 × (17 − 1 × 9) = 2 × 9 − 1 × 17

Substitute 9 = 3120 − 183 × 17:
1 = 2 × (3120 − 183 × 17) − 1 × 17
1 = 2 × 3120 − 366 × 17 − 1 × 17
1 = 2 × 3120 − 367 × 17

Therefore: d = −367, k = 2

📐 Step 3 — Make d Positive

Since d = −367 is negative, add φ(n) = 3120 to bring it into the positive range:

d = −367 + 3120 = 2753

Verification:

17 × 2753 = 46801
46801 ÷ 3120 = 15 remainder 1
∴ 46801 ≡ 1 (mod 3120) ✅
The Extended Euclidean Algorithm is the standard method for finding d in RSA key generation. Expect a step-by-step numerical question in exams.
📝 Common Exam Questions

  • Short: What equation does the Extended Euclidean Algorithm solve in RSA? → d × e ≡ 1 (mod φ(n))
  • Short: Why do we add φ(n) when d comes out negative? → Because we're working in modular arithmetic; negative values are converted to their positive equivalent mod φ(n)
  • Short: What must GCD(e, φ(n)) equal for the algorithm to work? → 1 (they must be coprime)

RSA Security & The Factoring Problem

📖 Why Is RSA Secure?

RSA's security rests entirely on the computational difficulty of factoring large integers. Given n = p × q (where p and q are large primes), there is no known efficient algorithm to factor n back into p and q when n is sufficiently large.

🔍 Three Approaches to Break RSA (Cryptanalysis)

ApproachMethodEquivalent To
Factor n Factor N = p × q, then compute φ(n) and find d Directly breaks RSA
Determine φ(n) directly Find φ(n) without factoring n, then find d Believed equivalent to factoring
Find d directly Determine the private key d without φ(n) Believed equivalent to factoring

All three approaches are currently believed to be equivalent in difficulty to factoring n — which is computationally infeasible for large n.

📈 State of Factoring (as of the slides)

  • As of Aug 1999: best factored was 130 decimal digits (512 bits) using GNFS (Generalized Number Field Sieve)
  • Biggest improvements came from algorithm improvements: Quadratic Sieve → Generalized Number Field Sieve
  • Conclusion: 1024+ bit RSA is secure against factoring attacks (barring dramatic breakthroughs)

⚙ RSA Key Size Recommendations

  • Ensure p and q are of similar size and satisfy certain constraints
  • Typical historical size: 1024 bits
  • Modern recommended minimum: 2048 bits
  • High-security environments: 4096 bits

Larger key = more security but slower performance. Trade-off must be balanced.

📝 Common Exam Questions
  • Short: What is the mathematical hardness problem that RSA relies on? → Integer factorization
  • Short: Name three ways an attacker could try to break RSA.
  • MCQ: What is the best practical factoring algorithm? → Generalized Number Field Sieve (GNFS)
  • Short: Why is 1024-bit RSA considered secure? → Factoring a 1024-bit number is computationally infeasible with current technology
  • Short: What constraint should p and q satisfy for security? → They should be of similar size

Timing Attacks on RSA

📖 What Is a Timing Attack?

A timing attack is a side-channel attack developed in the mid-1990s. Instead of attacking the mathematical structure of RSA, it exploits variations in the time taken to perform cryptographic operations. By carefully measuring how long decryption takes, an attacker can infer information about the private key.

⚡ How Timing Attacks Work

  • Multiplying by a small number takes less time than multiplying by a large number
  • Different execution paths (IF branches) cause different instruction counts, leading to different timing
  • Attacker measures timing of many decryption operations and uses statistical analysis to infer operand size
  • In RSA: exploits time taken during the modular exponentiation step (computing Cᵈ mod n)

🛡 Countermeasures

  • Constant Exponentiation Time: Ensure all exponentiations take the same time regardless of key bits (pad or delay)
  • Add Random Delays: Insert random pauses to break correlation between timing and key values
  • Blinding: Blind (randomize) values used in calculations before performing modular exponentiation, then unblind after — this masks actual timing from the attacker
Timing attacks are NOT about breaking the math of RSA — they are about leaking information through the physical implementation. Real-world RSA implementations MUST include timing attack countermeasures.

  • Short: When were timing attacks developed? → Mid-1990s
  • Short: What type of attack is a timing attack? → Side-channel attack
  • MCQ: Which step of RSA do timing attacks exploit? → Modular exponentiation
  • Short: Name three countermeasures against timing attacks.
  • Comparison: Compare a timing attack to a mathematical cryptanalysis attack on RSA.
