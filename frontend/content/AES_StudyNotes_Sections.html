01

Why AES Was Needed — DES Weakness & Multiple-DES

⚠ DES Weakness: Brute Force

DES is vulnerable to brute-force attack due to its small key size.

2⁵⁶ = 72,057,594,037,927,936 possible keys
The number of different possible keys in the obsolete 56-bit DES symmetric cipher.

Once the necessary technology became available, DES was cracked in a relatively short time after over 20 years of use as the encryption standard. NIST therefore decided a new standard — the Advanced Encryption Standard (AES) — was needed.

As an interim alternative that reuses existing DES software/equipment/knowledge, two options emerged:

  • Double DES
  • Triple DES (3DES)

⚙ Double DES

Double DES uses 2 × 56-bit keys, giving a nominal key length of 112 bits.

C = E(K2; E(K1; P))
Plaintext P is encrypted with K1, then encrypted again with K2.

However, the Meet-in-the-Middle attack makes Double DES much weaker than its 112-bit key size suggests.

🔴 Meet-in-the-Middle Attack

The attacker knows two known plaintext–ciphertext pairs (Pₐ, Cₐ) and (P_b, C_b).

  1. Encrypt Pₐ using all 2⁵⁶ values of K1 → generates multiple values of the intermediate value X. Store results in a table sorted by X.
  2. Note that E(K1; P) = X = D(K2; C).
  3. Decrypt Cₐ using all 2⁵⁶ values of K2.
  4. As each decryption result is produced, check it against the stored table.
  5. If a match is found, verify the candidate key pair (K1, K2) against the second pair: check whether encrypting P_b with these keys yields C_b.
  6. If P_b is obtained, accept the keys.
Encrypt/decrypt operations required: 2⁵⁶
(twice as many as single DES, NOT 2¹¹²)

With two known plaintext–ciphertext pairs, the probability of a successful attack is almost 1. This means Double DES provides no more security than single DES.

⚙ Triple DES (3DES)

Because Double DES is no more secure than single DES (due to the Meet-in-the-Middle attack reducing its work factor to roughly that of DES), 3DES was introduced. It uses 48 rounds in its computation, making it highly resistant to differential cryptanalysis.

3DES can take up to three times longer than DES to perform encryption and decryption — a heavy performance hit.

AES Overview & Background

📖 What Is AES?

The Advanced Encryption Standard (AES) is a symmetric key, symmetric block cipher standardised by NIST. It was created in 2001 after an open selection process. The winning algorithm, Rijndael, was developed by two Belgian cryptographers:

  • Vincent Rijmen
  • Joan Daemen

AES is stronger and faster than Triple-DES — approximately six times faster. It is the most widely adopted symmetric encryption algorithm and is generally considered secure.

✅ Strengths of AES

  • Implemented in both hardware and software — most robust security protocol.
  • Uses higher key sizes (128, 192, 256 bits) — makes it much more resistant to hacking than DES.
  • Most common security protocol used for wireless communication, financial transactions, e-business, encrypted data storage, and more.
  • One of the most widespread commercial and open-source solutions used worldwide.
  • For a 128-bit key, approximately 2¹²⁸ attempts are needed to break it — extremely difficult to hack.

AES Process Overview — Two Main Steps

📖 Two High-Level Steps

AES processing has two major components:

  1. Key Generation — Derives round sub-keys from the original key.
  2. Rounds — Processes the plaintext block through multiple rounds of transformation.

🔑 Key Generation Steps

  1. ROT word of the last column of the current key state — rotate the 4 bytes of the last column left by one position.
  2. SubByte of the rotated word — substitute each byte through the S-Box.
  3. XOR with RCON (round constant) and the first column of the key, then XOR with the SubByte result → becomes the first column of the new round key.
  4. Subsequent columns of the round key are derived by XOR-ing the previous column of the new key with the corresponding column of the previous key.

AES State & Key Representation

📖 The State Matrix

AES operates on a 4 × 4 matrix of bytes called the State. Both the plaintext block and the key are loaded into state matrices before processing begins.

  • The 128-bit (16-byte) message is arranged column-by-column into the 4 × 4 state matrix.
  • Example message: MESSAGEENCRPTION → each character occupies one byte cell in the matrix.
  • The 128-bit key (e.g., TEAMSCORPIAN1234) is similarly arranged into a 4 × 4 key state.
  • Every single column is one word = 4 bytes = 32 bits.
  • Each individual cell is 8 bits = 1 byte.

📐 State Dimensions Summary

UnitSize
Full State / Block128 bits = 16 bytes
One column (word)32 bits = 4 bytes
One cell8 bits = 1 byte
Matrix dimensions4 rows × 4 columns

🔑 Key Expansion into Round Keys

The original 128-bit key generates one round key per round plus one for the initial AddRoundKey. For a 128-bit key with 10 rounds, 11 round keys (round key 0 through round key 10) are generated from the original key using the key schedule.

Each round key is the same size as the block — 128 bits (4 words).

The sub-key generation process for each subsequent round key uses:

  1. Take the last column (word) of the current round key.
  2. Apply RotWord: rotate the 4 bytes left by one position.
  3. Apply SubWord: substitute each byte through the AES S-Box.
  4. XOR the result with the RCON (round constant — derived from powers of 2 in GF(2⁸)) for the current round.
  5. XOR the result with the first word of the previous round key → gives the first word of the new round key.
  6. Each subsequent word of the new round key = XOR of the previous word in the new key and the corresponding word in the old key.

Round Step 1 — AddRoundKey

📖 AddRoundKey Operation

AddRoundKey XORs the current State matrix with the corresponding round key matrix, byte by byte.

State[i][j] = State[i][j] ⊕ RoundKey[i][j]
⊕ = bitwise XOR, applied to every byte in the 4 × 4 state
  • Used in the Initial Round (with round key 0), every Main Round, and the Final Round.
  • It is the only operation that directly introduces key material into the cipher state.
  • The inverse of AddRoundKey is itself — XOR-ing with the same round key again undoes the operation.
💡 Intuition

Think of AddRoundKey as "mixing" secret key material into the data. XOR is symmetric — applying the same key twice returns the original, which is why encryption and decryption share this exact same step.

Round Step 2 — SubBytes

📖 SubBytes Operation

SubBytes is a non-linear byte substitution step. Every byte in the 4 × 4 State is independently replaced by looking it up in a fixed 16 × 16 S-Box table. The input byte is split into two hex digits: the high nibble (upper 4 bits) selects the row and the low nibble (lower 4 bits) selects the column.

  • The S-Box is constructed using the multiplicative inverse in GF(2⁸) followed by an affine transformation — making it highly non-linear.
  • Every byte is substituted independently — parallel operation on all 16 bytes.

🎯 Purpose

  • Provides confusion — makes the relationship between key and ciphertext as complex as possible.
  • The non-linearity of the S-Box is AES's primary defence against linear and differential cryptanalysis.
  • For decryption, use the inverse S-Box (InvSubBytes).

Round Step 3 — ShiftRows

📖 ShiftRows Operation

ShiftRows cyclically shifts each row of the State to the left by an offset equal to the row number.

RowLeft Shift (bytes)Effect
Row 00 — unchangedNo change
Row 11 byte leftFirst byte moves to end
Row 22 bytes leftFirst two bytes move to end
Row 33 bytes leftFirst three bytes move to end

🎯 Purpose

  • Provides diffusion — after ShiftRows, each column of the state contains bytes from four different original columns of S1.
  • This ensures the subsequent MixColumns step mixes bytes from all four columns, achieving complete inter-column diffusion within two rounds.
  • Decryption uses InvShiftRows — right shifts by the same amounts.

Round Step 4 — MixColumns & AddRoundKey

📖 MixColumns Operation

MixColumns treats each column of the State as a 4-element vector over GF(2⁸) and multiplies it by a fixed 4 × 4 matrix. Each output byte depends on all four input bytes of that column.

Fixed mixing matrix:
[ 02 03 01 01 ]
[ 01 02 03 01 ]
[ 01 01 02 03 ]
[ 03 01 01 02 ]
Multiplication over GF(2⁸). Addition = XOR. Values are hex.

🧮 GF(2⁸) Arithmetic Rules

  • Addition = bitwise XOR (⊕). Example: A3 ⊕ 1B = B8
  • ×01 = the byte unchanged.
  • ×02 = left-shift the byte by 1 bit. If the original high-order bit was 1, XOR the result with 0x1B to reduce it back to 8 bits.
  • ×03 = compute ×02, then XOR with the original byte.
  • Reducing polynomial: x⁸ + x⁴ + x³ + x + 1 (= 0x11B). Values above 0xFF are reduced by XOR with 0x1B.

⚠ MixColumns is omitted in the Final Round

This is intentional — omitting it makes the cipher structure symmetric for decryption. The inverse is InvMixColumns, which uses a different fixed matrix.

AES Encryption vs. Decryption

📖 Decryption — Inverting AES

AES decryption applies the inverse of each operation in reverse order. The round keys are applied in reverse (from round key Nᵣ down to round key 0).

🔄 Inverse Operations

Encryption StepDecryption InverseNotes
AddRoundKeyAddRoundKey (same)Self-inverse — XOR twice = identity
SubBytesInvSubBytesUses the inverse S-Box lookup table
ShiftRowsInvShiftRowsRight-shifts rows by 0, 1, 2, 3
MixColumnsInvMixColumnsUses a different fixed inverse matrix; omitted in final decryption round

🔬 Equivalent Inverse Cipher

AES also has an Equivalent Inverse Cipher formulation (as defined in the AES standard). In this variant, the order of InvShiftRows and InvSubBytes is swapped, and the AddRoundKey and InvMixColumns steps are interchanged. This is mathematically equivalent to the standard decryption but allows an implementation that mirrors the encryption structure more closely, potentially simplifying hardware implementations.

📝 Common Exam Questions
  • Which AES operation is its own inverse? → AddRoundKey
  • What is InvSubBytes? → Byte substitution using the inverse S-Box
  • What is InvShiftRows? → Cyclic right shift of rows by 0, 1, 2, 3 bytes
  • What is the Equivalent Inverse Cipher? → An alternative decryption formulation that swaps the order of some inverse operations for implementation convenience

AES Applications & Real-World Use

🌍 Where AES Is Used

AES is integral to the security framework of modern society. Key applications include:

  • File compression/encryption: RAR, WinZip
  • VPNs (Virtual Private Networks)
  • IEEE 802.11i — Wi-Fi Protected Access (WPA2) for wireless network security
  • Signal Protocol — the cryptographic protocol underlying secure messaging
  • Facebook Messenger (end-to-end encrypted messages)
  • WhatsApp (end-to-end encryption)
  • Wireless communication, financial transactions, e-business, encrypted data storage

✅ Why AES Is Trusted

  • Open, publicly vetted selection process — Rijndael won over 15 competing submissions.
  • Designed by academics with full public scrutiny for years before standardisation.
  • No practical attack has broken AES-128, AES-192, or AES-256 to date.
  • Approved by the NSA for Top Secret data when used with 192 or 256-bit keys.
  • Both hardware-accelerated (AES-NI CPU instructions) and efficient in software.
📝 Common Exam Questions
  • Name four real-world applications of AES.
  • Which wireless security standard uses AES? → IEEE 802.11i (WPA2)
  • Which messaging apps use AES-based encryption? → WhatsApp, Facebook Messenger (Signal Protocol)
